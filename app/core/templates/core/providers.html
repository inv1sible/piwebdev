Provider & Model Catalog

This page has two purposes: the model catalog (what appears in dropdowns) and a quick way to import Ollama models.
Authentication (API keys) is managed separately — see below.

How to add a new provider (e.g. Gemini, OpenAI, Anthropic…)

Step 1 — Set your API key
Add the key to /var/opt/piwebdev/.env and restart (docker compose restart web):

GEMINI_API_KEY=your-key-here
ANTHROPIC_API_KEY=your-key-here
OPENAI_API_KEY=your-key-here
XAI_API_KEY=your-key-here
DEEPSEEK_API_KEY=your-key-here

Alternatively, put keys in ~/.pi/agent/auth.json on the host (pi reads this directly).

Step 2 — Add provider + models to the catalog below
Use the Quick-add section to add a provider and model ID, then Save. The model will appear in the project settings dropdowns.

Step 3 — Select the provider in your project
Open a project → Pi settings → choose the provider and model → Save.

Saved.
PI_MODELS_JSON_PATH is not set.
Add the following to your .env file and restart the web container: 
PI_MODELS_JSON_PATH=/var/opt/piwebdev/models.json
